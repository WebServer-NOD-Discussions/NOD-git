Frequently Asked Questions

  1. How can the sequence of the protein of interest to be submitted to NOD be obtained??

    If the sequence of the protein of interest is already available in any publicly available databases like UniProt, it can be downloaded in .FASTA format and submitted to NOD. If it is unavailable, then, the experimental sequencing techniques need to be employed to obtain the sequence and then it can be submitted to NOD.


  2. What happens if only one sequence is submitted under MODE-1 operation of NOD?

    The MODE-1 of NOD is optimized for handling multiple sequences. Therefore, extensive analysis pertaining to expansion of the chemical space through similarity search method is not implemented in MODE-1 which would be highly time-consuming when dealing with a large input set of protein sequences. Whereas MODE-2 is optimized for handling only a single input sequence and is therefore faster. Additional feature of expansion of chemical space through chemical similarity search is available only in MODE-2. Thus, if a single sequence is given as input in MODE-1, the user will be deprived of the extensive analysis features available in MODE-2.


  3. What happens if multiple sequences are submitted under MODE-2 operation of NOD?

    MODE-2 of NOD accepts only one sequence per job. The algorithm will throw an error and the job will be terminated if more than one sequence is submitted under MODE-2.


  4. How long does NOD take to run a job?

    On an average a single sequence takes ~1 second to complete the Jackhmmer search. The time taken for post-processing the Jackhmmer results to identify reliable hits and fetch the associated DrugBank molecules varies from case-to-case basis. The total number of Jackhmmer hits obtained from homology search, the number of known DrugBank molecules for those hits are the two major factors which determine the run time of the jobs. For example, a submitted sequence which is homologous to any kinases or GPCRs, might take a longer time to run as the number of DrugBank molecules against these proteins are higher. Therefore, the time taken in fetching details of each molecule against each protein hit is expected to be longer than the cases where the number of homologous protein hits and associated DrugBank molecules are less.


  5. How to interpret the results obtained from NOD server?

    Descriptions of the data tabulated in each column of the result table are explained under the ‘How to read the Table’ section in the html result file (the link to which is sent in the e-mail to the user). Sample result files with interpretation can be found here.


  6. What is the confidence level that the predicted repurpose-able candidates will translate to similar observations during in vitro and in vivo experiments?

    Any computational pipeline for drug repurposing approaches are the initial points which help to rationally narrow down the search space by providing a reasonably small set of compounds to start with. Careful assessment of the results from our computational drug discovery pipelines have shown agreement with experimental results in the past which proves the strength of our protocol. It is therefore expected that results from the NOD pipeline when probed rationally will help the users to take informed decisions and plan subsequent experiments which are expected to corroborate with experimental observations.


  7. Are there any published literature which can be referred to as a guide for in-depth analysis of the results obtained from NOD?

    Interested users are encouraged to read the relevant publications from our group which can be found here. The mentioned publications demonstrate the use of pipelines similar to NOD and integration of structure-based analytical approaches to extract meaningful insights from sequence-based analyses. These can serve as guide to non-experts to get an idea on planning subsequent analysis.


  8. Can NOD accept a DNA and RNA sequence?

    No, NOD does not accept DNA and/RNA sequences. Only protein sequences should be submitted.


  9. Can NOD accept any in-house protein sequence which is not available in the public domain?

    Yes, NOD can accept any valid protein sequence (irrespective of its availability in public domain). Ideally, NOD should reformat the input query into NOD-friendly .FASTA format. If an error still occurs, please report the error to us.


  10. An e-mail notification regarding job completion on NOD server has not been received. What can be done?

    The SMTP server might have detected an error in the user’s email domain and pushed the email from NOD in the Spam/Junk folder. NOD users are requested to check the Spam/Junk folder if they are unable to find any email in the primary inbox. To avoid missing emails from NOD, users may wish to add nod.server.pushemail@gmail.com to their list of trusted contacts. If the email is not found even in the Spam/Junk folder, users are requested to wait as it implies that the submitted job is not yet completed. However, if a user finds that the wait time is unusually long, the NOD team can be contacted by filling up the contact form with relevant details.


  11. Can NOD be used for commercial purposes?

    NOD can be used freely by academic as well as commercial users unless proper citation to NOD is done.


  12. Can NOD algorithm be used as a standalone program on the local system of the users?

    As of now, NOD is only available as a web-based pipeline. The NOD team is working toward making this pipeline available as a standalone version in the future.


  13. Can anyone consume the listed drugs obtained as hits in NOD result to get rid of a disease in which the protein of interest is implicated?

    NO, Team NOD strongly discourages the practice of self-medication. The NOD server is solely intended to aid the scientific community for research activities related to drug repurposing programs. The results obtained from this server shall not be interpreted for any medical advice, diagnosis or treatment without approval from any competent regulatory/medical authority and subsequent consultation with registered medical practitioners.


  14. What are the limitations of NOD?

    NOD is primarily a sequence-based homology search mediated pipeline to identify potential drugs/drug candidates which can be repurposed against the disease of interest. The pipeline ensures that only reliable associations are fetched (query coverage: ≥ 70% of the alignment length and E-value < 10-4) for further analysis.
    So, it is likely that ligand binding regions of a pair of homologous proteins would be similar and by virtue of this similarity, it can be expected to recognize similar ligands. However, if the alignment region does not encompass the ligand binding region, then the chances of recognizing similar molecules is low.
    Users are encouraged to inspect the alignment files available with result table to confirm if the aligned regions of the protein sequences span over the ligand binding region. Further, ligand binding is an event that involves the 3-D coordinate space, which is influenced by a bunch of other physiological factors (like shape and electrostatic feature of the ligand binding pocket, influence of the surrounding environment, flexibility of the protein and ligand etc.). These factors are beyond the scope of NOD to consider. Thus, the results from NOD shall be treated as a starting point for any drug repurposing projects and it is advisable that the user should perform thorough analysis on a case-to-case basis as recommended in Q7. Additionally, it should be noted that many DrugBank molecules which are listed against a particular protein target are not true drugs like nutraceuticals, dietary supplements, metallic elements approved for usage in medical devices etc. However, differentiating true drugs/drug candidates from molecules which are unlikely to be drugs is beyond the scope of NOD. Upon establishing a reliable evolutionary relationship between a query protein sequence and the dataset of target protein sequences, any DrugBank molecule linked with the later are fetched. Users are recommended to thoroughly analyze the details of each DrugBank hit. To make this process user-friendly, the hits in the result table of the html file are hyperlinked to the respective DrugBank page. Users should visit the respective DrugBank page to know details about each hit molecule.


  15. How can a user reach the NOD team if there are any queries?

    All users are requested to go through the FAQs thoroughly. In case, there is any query which is not addressed above, users may feel free to write to the NOD team by filling up the contact form which can be found here. The team shall get back to the user ASAP. Additionally, feedback and/suggestions from the users are always welcome which can also be submitted using the said form. The NOD team looks forward to learning from users’ experiences to improve further and provide better services. The team is thankful to all the NOD users.
